# MCP Integration (Claude Desktop / AI Agent Access)

## What Is MCP?
MCP (Model Context Protocol) is an open standard that lets Claude Desktop and other AI clients connect directly to Vibe Trial Balance. Once connected, Claude can read your data, answer questions about it, and take actions on your behalf — without you copying and pasting information back and forth.

## Two Connection Modes

### stdio — Local
Runs a Node.js process that connects directly to the database. Best for Claude Desktop running on the same machine as the app server (or with direct database access). No MCP token required.

Claude Desktop config (`claude_desktop_config.json`):
```json
{
  "mcpServers": {
    "trial-balance": {
      "command": "node",
      "args": ["/path/to/vibe-tb/server/dist/mcp-stdio.js"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost:5432/trial_balance",
        "ANTHROPIC_API_KEY": "sk-ant-..."
      }
    }
  }
}
```
Update the path and database credentials before use.

### HTTP/SSE — Remote
Connects over the network using Server-Sent Events. Best for Claude Desktop on a different machine than the server. Requires an MCP token generated by an admin.

Claude Desktop config:
```json
{
  "mcpServers": {
    "trial-balance-http": {
      "url": "http://your-server:3001/mcp/sse",
      "headers": {
        "Authorization": "Bearer <your-mcp-token>"
      }
    }
  }
}
```

## Token Management (Admin Only)
Go to **Settings** (Tools section of the sidebar). Scroll to the **MCP / Claude Desktop Integration** card.

- **Generate Token**: Creates a new token. The full token is shown exactly once — copy it immediately before navigating away.
- **Rotate Token**: Replaces the existing token. The old token is immediately invalidated.
- **Revoke Token**: Removes the token entirely and disconnects all active HTTP/SSE clients.

The Settings page shows ready-to-copy configuration snippets for both stdio and HTTP/SSE modes.

## What Claude Can Read (Resources)
Once connected, Claude can read:
- All clients and their details
- All periods for a client (with lock status)
- Adjusted trial balance for a period (with balance check)
- Chart of accounts with tax code assignments
- Journal entries for a period
- Period dashboard stats and recent audit activity

## What Claude Can Do (Tools)

### Data Retrieval
- **list_clients** — list all clients in the system
- **get_period_dashboard** — key stats and recent activity for a period
- **get_trial_balance** — adjusted TB, optionally filtered by category (assets/liabilities/equity/revenue/expenses)
- **list_journal_entries** — JEs for a period, optionally filtered by type (book/tax)
- **get_chart_of_accounts** — COA with optional filters (category, unmapped only, search text)
- **get_tax_mapping_status** — mapping progress percentage and unmapped account count
- **get_period_comparison** — dollar and percentage variances between two periods
- **list_engagement_tasks** — checklist tasks for a period, optionally filtered by status
- **search_accounts** — search COA by account name or number
- **generate_report_url** — returns the URL to download a specific PDF report

### Actions (Write Operations)
- **update_trial_balance_row** — update unadjusted debit/credit balance for an account (period must be unlocked)
- **create_journal_entry** — create a book or tax AJE with balanced lines (period must be unlocked)
- **auto_assign_tax_codes** — run the 5-step AI waterfall to suggest tax code assignments for unmapped accounts
- **confirm_tax_assignments** — write confirmed tax code assignments to the database (dual-writes tax_line for compatibility)
- **run_diagnostics** — run AI-powered diagnostics (balance check, variance analysis, completeness check)
- **lock_period** — lock a period (TB must be balanced first)
- **update_engagement_task** — update a task's status (open/in_progress/review/completed/n_a) or notes
- **add_variance_note** — add or update a variance explanation note for an account

## Pre-Built Workflows (Prompts)
Use these in Claude Desktop to start a structured multi-step workflow:

| Prompt | What It Does |
|---|---|
| **period_review** | Dashboard → TB balance check → AI diagnostics → summary |
| **tax_mapping_review** | Check mapping status → auto-assign → confirm high-confidence results |
| **variance_investigation** | Compare two periods → identify significant variances → add notes |
| **engagement_status** | Summarize tasks by category, flag overdue and high-priority items |
| **full_period_close** | Complete close: dashboard → diagnostics → TB → tax mapping → engagement → lock |

Example: In Claude Desktop, type "Run the period_review prompt for period 5" and Claude will execute all steps automatically.

## Security and Audit Logging
- All tool calls are recorded in the **Audit Log** (Admin > Audit Log), attributed to the `mcp_agent` system user
- Rate limiting: 100 tool calls per minute per connection
- Token management requires admin role
- The `mcp_agent` user cannot log in through the web UI — it exists solely to attribute MCP actions in the audit log

## Typical Use Cases
- **"Show me the current trial balance for Demo Company FY2024"** — Claude reads the TB and presents a summary
- **"Which accounts don't have tax codes assigned?"** — Claude checks the COA for unmapped accounts
- **"Run diagnostics on period 3"** — Claude checks balance, analyzes variances, flags issues
- **"Auto-assign tax codes for client 1 and confirm the high-confidence ones"** — Claude runs the AI waterfall and confirms assignments ≥80% confidence
- **"Close period 5"** — Claude runs the full_period_close workflow: checks balance, runs diagnostics, reviews tax mapping, then locks
