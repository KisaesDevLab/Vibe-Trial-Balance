# Custom Reports

## Overview
The Custom Report Builder lets you create saved report layouts tailored to your firm's needs or a specific client's reporting requirements. Access: **Reports > Custom Reports**

## What Custom Reports Can Do
- Select which accounts appear in the report
- Choose which balance columns to display (unadjusted, book-adjusted, tax-adjusted, prior year)
- Group accounts by category with subtotals
- Save the layout and reuse it across periods

## Creating a Custom Report
1. Go to **Reports > Custom Reports**
2. Click **New Report**
3. Enter a report name (e.g., "Management Report — Cash Basis", "Partner Capital Summary")
4. Configure the sections:
   - Add one or more **sections**, each with a title
   - For each section, select which accounts to include (search or browse by category)
   - Choose the balance columns for each section
5. Click **Save**

## Running a Saved Report
1. Select your client and period in the sidebar
2. Go to **Reports > Custom Reports**
3. Click any saved report to view it with the current period's data
4. The report renders with the accounts and columns you configured

## Updating a Report Layout
Click the edit icon on any saved report to modify its sections, accounts, or column selection. Changes apply immediately when you next run the report.

## Deleting a Report
Click the delete icon on any saved report. This removes the saved layout — it does not affect any account data.

## Saved Reports and Periods
Report layouts are saved at the client level and work across all periods for that client. The same layout applied to FY2024 and FY2025 will show each period's data when you switch periods in the sidebar.

## Tips
- Use custom reports for non-standard groupings that the built-in financial statements don't cover (e.g., a report showing only operating accounts, or a schedule of fixed assets)
- Custom reports are good for client-facing deliverables that need a specific format
- Combine with the PDF download button to produce branded PDF versions
- For standard financial statements (IS, BS, Cash Flow), use the built-in reports under **Reports > Financial Statements** instead
